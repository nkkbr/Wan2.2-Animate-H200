# Step 12: 参考图尺度归一化

## 1. 目标

让参考全身图在进入 replacement 系统前，先与驱动视频中的人物尺度、构图和人体占比建立更合理的对齐关系，从而降低体型错配和肢体变形。

这一步的目标是：

- 减少参考人物与驱动人物体型差异带来的违和感
- 改善角色在视频画幅中的尺度先验
- 降低后续生成对静态参考图的“硬投射”压力


## 2. 为什么把它放在最后

这一步有明确价值，但不属于最核心的基础设施。

它应放在后面，原因有三：

1. 当前更大的问题仍是控制信号质量和 continuity。
2. 参考图尺度归一化更多是“进一步抬高上限”，不是修复底层 correctness。
3. 它容易和风格、服装、姿态差异纠缠在一起，验证成本高。


## 3. 本步骤的范围

本步骤包括：

- 参考图人体检测
- 参考图人物 bbox / skeleton 估计
- 参考图缩放与画布放置策略
- 与驱动视频第一段或统计尺度的对齐

本步骤不做：

- 多参考图融合
- 重新设计 CLIP token 结构
- 新的模型训练


## 4. 当前问题

当前 `src_ref.png` 在 generate 中基本是：

- 直接读图
- letterbox 到视频尺寸

这意味着：

- 参考图中的人体占比可能与驱动视频完全不一致
- 单张竖版全身图会被硬塞进横版视频画布
- 模型需要自己同时解决身份迁移和尺度对齐

当参考人物与原人物体型差异较大时，这会放大：

- 肢体变形
- 衣物体积不自然
- 头身比不协调


## 5. 设计原则

### 5.1 先做参考图尺度归一化，不动模型结构

优先通过输入预处理改善问题，而不是直接改主干。

### 5.2 对齐的是“统计尺度”，不是强制姿态一致

replacement 模式当前默认不做 pose retarget。  
因此这一步不应试图把参考图姿态改成驱动姿态，而是：

- 对齐人物占比
- 对齐画面摆放

### 5.3 保留可关闭开关

有些 case 原始参考图已经合适。  
因此这个步骤应是可选增强，而不是强制路径。


## 6. 具体实施方案

### 6.1 参考图人体检测

建议对参考图先做：

- 人体 bbox 检测
- 可选骨架估计

目标是得到：

- 人物在参考图中的占比
- 人体中心位置
- 大致 body scale

### 6.2 驱动视频目标尺度估计

建议从驱动视频中估计一个目标尺度，可选方式：

- 第一帧人物 bbox
- 前若干帧平均 bbox
- 全视频统计中位数 bbox

第一版建议用前若干帧的中位数。

### 6.3 参考图重排版

根据目标尺度，对参考图做：

- 缩放
- 平移
- padding

输出一个更贴近驱动视频人物占比的 `src_ref_normalized.png`。

### 6.4 metadata 记录

建议 metadata 写清：

- 是否启用 reference normalization
- 使用了哪种目标尺度估计
- 缩放因子
- 画布位置


## 7. 需要改动的文件

建议涉及：

- preprocess 参考图处理逻辑
- metadata schema
- `wan/animate.py` 的参考图读取部分

必要时新增：

- reference normalization 工具模块


## 8. 如何实施

建议顺序如下：

1. 先做参考图人体 bbox 检测。
2. 再做驱动视频目标尺度估计。
3. 再做参考图缩放与排版。
4. 最后在 generate 中切换为读取 normalized reference。


## 9. 如何检验

### 9.1 人物占比检验

检查 normalized reference 与驱动视频人物占比是否更接近。

### 9.2 replacement 结果检验

重点看：

- 肢体比例
- 服装体积感
- 角色在画面中的自然程度

### 9.3 消融实验

至少比较：

- 原始 reference
- normalized reference

在体型差异明显的 case 上做重点验证。


## 10. 验收标准

只有满足下面条件，才算本步骤完成：

- normalized reference 的人物占比更接近驱动视频
- benchmark 中体型差异大的 case 明显更自然
- 不会普遍伤害原本已经正常的 case


## 11. 风险与回退

### 风险

- 对某些 case 误判目标尺度
- 过度归一化，让参考图失去原本的构图优势

### 回退策略

- 该功能默认可关闭
- 支持多种目标尺度估计方式
- 仅在体型差异明显的素材上优先启用


## 12. 本步骤完成后的收益

这一步更像“上限增强项”。  
当基础控制信号与 continuity 已经足够稳时，它能进一步减少 replacement 中最常见的人体比例违和感。
